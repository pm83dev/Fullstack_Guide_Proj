Backup di trigger, disabilitazione, OBJECT_DEFINITION, REPLACE del nome
Strategia — Backup prima di Modificareworkflow
Procedura sicura
Prima di modificare un trigger in produzione: 1) crea una copia di backup rinominata _OLD, 2) disabilita il backup così non interferisce, 3) modifica il trigger originale. In caso di errore puoi abilitare la copia e confrontarla.
Script — Backup e Disabilitazione Triggersql
USE [PIM_Test];  -- sostituisci con il tuo DB
GO

-- 1. Crea la copia di backup del trigger
IF OBJECT_ID('dbo.CatalogBaseProduct_chktrg_OLD', 'TR') IS NOT NULL
BEGIN
    PRINT 'Trigger di backup già esistente.';
END
ELSE
BEGIN
    DECLARE @triggerDefinition NVARCHAR(MAX);

    -- Recupera il testo del trigger originale
    SELECT @triggerDefinition = OBJECT_DEFINITION(
        OBJECT_ID('dbo.CatalogBaseProduct_chktrg')
    );

    -- Rinomina nel testo per creare il backup
    SET @triggerDefinition = REPLACE(
        @triggerDefinition,
        'CatalogBaseProduct_chktrg',
        'CatalogBaseProduct_chktrg_OLD'
    );

    -- Esegue il codice per creare il trigger di backup
    EXEC(@triggerDefinition);

    PRINT 'Trigger di backup creato: CatalogBaseProduct_chktrg_OLD';
END
GO

-- 2. Disabilita la copia di backup
IF OBJECT_ID('dbo.CatalogBaseProduct_chktrg_OLD', 'TR') IS NOT NULL
BEGIN
    DISABLE TRIGGER dbo.CatalogBaseProduct_chktrg_OLD
    ON dbo.CatalogBaseProduct;
    PRINT 'Trigger di backup disabilitato.';
END
GO

-- 3. Il trigger originale resta attivo e pronto per modifiche
PRINT 'Trigger originale pronto per modifiche: CatalogBaseProduct_chktrg';
GO
Comandi Utili sui Triggerquick ref
-- Visualizza il testo di un trigger
SELECT OBJECT_DEFINITION(OBJECT_ID('dbo.NomeTrigger'));

-- Lista tutti i trigger su una tabella
SELECT name, is_disabled, type_desc
FROM sys.triggers
WHERE parent_id = OBJECT_ID('dbo.NomeTabella');

-- Disabilita un trigger
DISABLE TRIGGER dbo.NomeTrigger ON dbo.NomeTabella;

-- Abilita un trigger
ENABLE TRIGGER dbo.NomeTrigger ON dbo.NomeTabella;

-- Elimina un trigger
DROP TRIGGER IF EXISTS dbo.NomeTrigger;

-- Verifica se un trigger esiste
IF OBJECT_ID('dbo.NomeTrigger', 'TR') IS NOT NULL
    PRINT 'Trigger esiste';
OBJECT_DEFINITION + EXEC — il trucco per clonare
OBJECT_DEFINITION(OBJECT_ID(...)) restituisce il DDL completo del trigger come stringa. Con REPLACE() cambi il nome e con EXEC() esegui la stringa per creare il clone. Funziona anche per stored procedure e view.
