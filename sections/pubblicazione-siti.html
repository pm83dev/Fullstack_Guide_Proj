PARTE A — Sito HTML Statico su IISIIS
# A2 — Configurare IIS Manager
Sites → Add Website:
  Site name:     testsito
  Physical path: C:\inetpub\wwwroot\testsito
  Port:          8080
  Host name:     ← lasciare VUOTO (importante!)

# Verificare che risponda in locale
curl -v http://localhost:8080
⚠ Host name DEVE essere vuoto
Se si inserisce un hostname in IIS, le richieste dal tunnel Cloudflare vengono rifiutate con errore 400 Invalid Hostname.
# A3 — Cloudflare Dashboard → Zero Trust → Networks → Tunnels
# → Edit → Public Hostname → Add a public hostname:
Subdomain: testsito
Domain:    pm-softwareautomation.com
Type:      HTTP
URL:       http://localhost:8080

# config.yml del tunnel (C:\Users\pc\.cloudflared\config.yml)
tunnel: <tunnel-id>
credentials-file: C:\Users\pc\.cloudflared\<tunnel-id>.json
ingress:
  - hostname: testsito.pm-softwareautomation.com
    service: http://localhost:8080
  - service: http_status:404

# Riavviare cloudflared (PowerShell)
Restart-Service cloudflared
# A4 — Verifica DNS Cloudflare
# Il record DEVE essere Proxied (nuvoletta arancione)
Resolve-DnsName testsito.pm-softwareautomation.com -Server 1.1.1.1
# Deve restituire indirizzi A IPv4 di Cloudflare (es. 188.114.96.7)

# Test finale
curl -v https://testsito.pm-softwareautomation.com
PARTE B — App Docker tramite IIS Reverse ProxyDocker + IIS

Prerequisiti: Docker Desktop installato, IIS con moduli ARR (Application Request Routing) e URL Rewrite installati da iis.net.

# B3 — docker-compose.yml dell'app
services:
  app:
    image: mia-app
    restart: unless-stopped
    ports:
      - "5100:80"
    environment:
      - ASPNETCORE_ENVIRONMENT=Production

docker compose up -d
curl -v http://localhost:5100   # deve rispondere 200 OK
# B4 — IIS: crea sito come reverse proxy
Sites → Add Website:
  Site name:     demo-mia-app
  Physical path: C:\inetpub\wwwroot\demo-mia-app  (cartella vuota)
  Port:          8081
  Host name:     ← VUOTO

# Configura URL Rewrite come reverse proxy
# Seleziona il sito → URL Rewrite → Add Rule → Reverse Proxy
Server name: localhost:5100
# Abilita SSL Offloading → OK

# web.config generato automaticamente
<?xml version="1.0" encoding="UTF-8"?>
<configuration>
  <system.webServer>
    <rewrite>
      <rules>
        <rule name="ReverseProxy" stopProcessing="true">
          <match url="(.*)" />
          <action type="Rewrite" url="http://localhost:5100/{R:1}" />
        </rule>
      </rules>
    </rewrite>
  </system.webServer>
</configuration>
# Verifica reverse proxy
curl -v http://localhost:8081
# Risposta identica a localhost:5100 — IIS sta agendo da proxy
# B5 — Cloudflare: aggiungi Public Hostname
Subdomain: demo-mia-app
Domain:    pm-softwareautomation.com
Type:      HTTP
URL:       http://localhost:8081   ← porta IIS, non Docker!

# config.yml aggiornato (aggiungi PRIMA della riga http_status:404)
ingress:
  - hostname: testsito.pm-softwareautomation.com
    service: http://localhost:8080
  - hostname: demo-mia-app.pm-softwareautomation.com
    service: http://localhost:8081   ← IIS proxy verso Docker
  - service: http_status:404

Troubleshooting — Errori Comunidebug
SintomoCausa e Soluzione
DNS_PROBE_FINISHED_NXDOMAINRecord DNS non propagato. Verificare con Resolve-DnsName <host> -Server 1.1.1.1
400 Invalid HostnameBinding IIS ha Host name specifico → impostare a vuoto. O rimuovere httpHostHeader dal config.yml
Tunnel Status: Downcloudflared non in esecuzione. Eseguire Restart-Service cloudflared
Solo IPv6, nessun IPv4Record DNS non è Proxied (nuvoletta grigia) → cambiarlo in Proxied su Cloudflare
IIS binding su 127.0.0.1Tunnel non riesce a connettersi → cambiare binding da 127.0.0.1 a All Unassigned (*)
