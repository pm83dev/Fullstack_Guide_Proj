Quando NON conviene
Se usi software industriale, CAD, tool vendor-specific per PLC/SCADA su Windows, WSL2 è la soluzione ideale: sviluppo Linux + Windows per tutto il resto.
Setup Iniziale WSL2verifica
# Da PowerShell — verifica WSL2 installato
wsl --list --verbose

# Output atteso:
  NAME      STATE    VERSION
* Ubuntu    Stopped  2

# Installare Ubuntu se non presente
wsl --install -d Ubuntu

# Reset password dimenticata
wsl -u root
passwd nomeutente
exit
Nota
Le distro docker-desktop e docker-desktop-data sono interne a Docker — non usarle per sviluppare. Serve una distro Ubuntu/Debian propria.

Dove mettere i progetti
Usa ~/progetti/... (filesystem Linux) per performance ottimali. /mnt/c/Users/... funziona ma ha I/O lento su build con molti file (node_modules).
Migrazione Progetto da Windows a WSLmigrazione
# Copia il progetto nel filesystem Linux
cp -r /mnt/c/Users/paolo/progetti/mio-progetto ~/progetti/
cd ~/progetti/mio-progetto

La Git history viene copiata integralmente. I remote URL rimangono validi.

# File .gitattributes — forza LF nel repository (standard Windows/Linux)
* text=auto eol=lf

# File .editorconfig
root = true
[*]
end_of_line = lf
charset = utf-8
trim_trailing_whitespace = true
insert_final_newline = true

# Normalizzare un repo esistente con CRLF
git add --renormalize .
git commit -m 'chore: normalize line endings to LF'

Problemi Noti Angular 21 + NGXSfix
Problema 1 — SSR abilitato di default
Angular 21 genera con Server Side Rendering abilitato. NGXS non è compatibile con SSR.
Soluzione: in angular.json → architect → build → options rimuovere le chiavi server, outputMode e ssr.
Problema 2 — Zone.js non installato
Angular 21 è zoneless di default. NGXS richiede zone.js.
Soluzione: npm install zone.js + aggiungere "polyfills": ["zone.js"] in angular.json.
Problema 3 — Naming convention cambiata
In Angular 21 il componente root si chiama App (non AppComponent) e il file è app.ts (non app.component.ts).
Problema 4 — CORS bloccato
Il browser blocca le richieste da localhost:4200 verso localhost:5035.
Soluzione: aggiungere la policy CORS in Program.cs (vedi sezione .NET → Program.cs).
Problema 5 — MapControllers mancante
Il template webapi usa Minimal API. Per i controller aggiungere in Program.cs:
builder.Services.AddControllers(); e app.MapControllers();
