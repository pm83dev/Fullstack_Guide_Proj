Template — Event Log con Visualizzazione Modifichetemplate
<div class="event-log-container">
  <h2>Event Log</h2>

  <div class="filters">
    <input [(ngModel)]="entityFilter"
           placeholder="Filtra per entità"
           (input)="loadEvents()" />
    <button (click)="loadEvents()">Aggiorna</button>
  </div>

  <div class="events-list" *ngIf="events.length > 0">
    <div class="event-item"
         *ngFor="let event of events; trackBy: trackByEventId"
         [class.created]="event.eventType === 'Created'"
         [class.updated]="event.eventType === 'Updated'">

      <div class="event-header">
        <span class="event-type">{{ event.eventType }}</span>
        <span class="event-date">{{ formatDate(event.createdAt) }}</span>
      </div>

      <div class="event-details">
        <p><strong>Azienda:</strong> {{ event.companyName }}</p>
        <p><strong>Modificato da:</strong>
           {{ event.userInfo?.userFullName || 'N/A' }}</p>
      </div>

      <!-- Mostra diff solo se evento Updated e ci sono modifiche -->
      <div class="changes"
           *ngIf="event.eventType === 'Updated' && hasChanges(event)">
        <h4>Modifiche:</h4>
        <ul>
          <!-- Modifiche azienda -->
          <li *ngFor="let change of event.newDataCompany ?? []"
              class="company-change">
            <strong>Company {{ change.fieldName }}:</strong>
            "{{ change.oldValue }}" → "{{ change.newValue }}"
          </li>

          <!-- Modifiche persone -->
          <li *ngFor="let person of event.newDataPerson ?? []"
              class="person-change">
            <div *ngIf="hasPersonChanges(person)">
              <strong>{{ person.name }} {{ person.surname }}</strong>
              <ul>
                <li *ngFor="let change of person.newDataPerson">
                  <strong>Person {{ change.fieldName }}:</strong>
                  "{{ change.oldValue }}" → "{{ change.newValue }}"
                </li>
              </ul>
            </div>
          </li>
        </ul>
      </div>

    </div>
  </div>

  <div class="no-events" *ngIf="events.length === 0 && !loading">
    Nessun evento trovato.
  </div>

  <div class="loading" *ngIf="loading">Caricamento eventi...</div>
</div>
Metodi del Componentetypescript
export class EventLogComponent {
  events: EventLog[] = [];
  entityFilter = '';
  loading = false;

  // trackBy per performance *ngFor
  trackByEventId(index: number, event: EventLog): string {
    return event.id;
  }

  // Controlla se l'evento ha modifiche da mostrare
  hasChanges(event: EventLog): boolean {
    return (event.newDataCompany?.length ?? 0) > 0
        || (event.newDataPerson?.length ?? 0) > 0;
  }

  hasPersonChanges(person: PersonChangeDto): boolean {
    return (person.newDataPerson?.length ?? 0) > 0;
  }

  formatDate(iso: string): string {
    return new Date(iso).toLocaleString('it-IT');
  }
}
Flusso — Manipolazione DOM in JavaScript Vanillaworkflow
Sequenza corretta per render dinamico
Seguire questo ordine evita duplicati, memory leak e race condition con gli event listener.
1. Definisci struttura HTML
      - Contenitori fissi (div, table, tbody)
      - ID/classi univoci per la selezione
        ↓
2. Scrivi i dati (array di oggetti)
        ↓
3. Seleziona elementi chiave
   document.getElementById('#id')
   document.querySelector('.class')
        ↓
4. Scrivi le funzioni di render
   renderCards(data)
   renderTable(data)
        ↓
5. Dentro ogni funzione:
      - Svuota contenitore (innerHTML = '')
      - Crea nodi o stringhe HTML
      - Appendi al DOM
        ↓
6. Aggancia eventi UNA SOLA VOLTA
   input.addEventListener('input', handler)
   button.addEventListener('click', handler)
        ↓
7. Event handler:
      - Leggi dati aggiornati (es: filtro corrente)
      - Chiama funzione di render
innerHTML = '' prima del render
Svuotare sempre il contenitore prima di aggiungere nuovi nodi. Altrimenti ogni chiamata a render() accumula elementi duplicati nel DOM.
