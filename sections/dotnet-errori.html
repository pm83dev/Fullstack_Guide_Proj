Angular 21 — SSR abilitato di defaultAngular 21

Angular 21 genera i nuovi progetti con SSR attivo. NGXS non è compatibile con SSR e causa errori Zone.js nel contesto server.

// angular.json — sezione options CORRETTA (senza SSR)
"options": {
  "browser": "src/main.ts",
  "polyfills": ["zone.js"],
  "tsConfig": "tsconfig.app.json",
  "inlineStyleLanguage": "scss",
  "assets": [{ "glob": "**/*", "input": "public" }],
  "styles": ["src/styles.scss"]
}
// Rimuovere: "server", "outputMode", "ssr"

Angular 21 è zoneless di default. NGXS richiede ancora zone.js per il change detection.

Angular 17+ — Naming convention cambiataAngular 17+

Il componente root non si chiama più AppComponent ma App. Gli import in main.ts devono usare il nuovo nome.

// ❌ SBAGLIATO (Angular < 17)
import { AppComponent } from './app/app.component';
bootstrapApplication(AppComponent, appConfig);

// ✅ CORRETTO (Angular 17+)
import { App } from './app/app';
bootstrapApplication(App, appConfig);

NGXS — Payload double-nesting (errore frequente)NGXS
// ❌ SBAGLIATO — il service riceve { payload: { OrdNum: 7 } }
@Action(GetDocumentsAction)
getDocuments(ctx, action: GetDocumentsAction) {
  return this.svc.getDocuments({ payload: action.payload }); // ← ERRORE
}

// ✅ CORRETTO — passa payload direttamente
@Action(GetDocumentsAction)
getDocuments(ctx, action: GetDocumentsAction) {
  return this.svc.getDocuments(action.payload); // action.payload = { OrdNum: 7 }
}
