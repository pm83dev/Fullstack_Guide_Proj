Procedura di Sviluppo Step-by-Steproadmap
StepCosa fare
1 — LM StudioScarica da lmstudio.ai, carica Qwen2.5 14B Q4, avvia server (porta 1234). Verifica: curl http://localhost:1234/v1/models
2 — Scaffold .NETCrea sln con 5 progetti (API, Application, Domain, Infrastructure, Worker)
3 — PostgreSQLdocker run -d --name sds-postgres -e POSTGRES_PASSWORD=devpassword -p 5432:5432 postgres:16-alpine + EF Core + DbContext
4 — JWT AuthJWT + Refresh Token Rotation. TenantId nei claims. Middleware che inietta TenantId nel scope.
5 — Pipeline PDF→LLMPython sidecar con pdfplumber (POST /extract-text). ILlmService per LM Studio. Salva JSON in JSONB.
6 — BackgroundWorkerChannel<Guid> + BackgroundService. Recovery al riavvio: riaccoda status=pending/processing.
7 — Angularng new, ng-bootstrap, AuthGuard, TenantInterceptor (JWT header), polling ogni 3s.
8 — Export Excelopenpyxl nel sidecar Python. Colonne dinamiche da JSONB.
9 — Docker ComposeTest end-to-end in container. Verifica recovery worker dopo restart.
10 — Deploy VPSSubdomain, Nginx, Certbot SSL. Switch lmstudio → anthropic in appsettings.Production.json
💡 Consiglio pratico
Inizia con Step 1-5 e testa la pipeline su 2-3 SDS reali prima di costruire il resto. Il prompt LLM è la parte più critica. Con LM Studio + Qwen2.5 14B aspettati 30-60 secondi per SDS: normale per dev/test.
