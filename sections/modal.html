Modal + ng-content
Componente generico riutilizzabile con @Input/@Output e Content Projection
🧠 Come funziona

modal.html fornisce la struttura (header, body, footer Bootstrap). <ng-content> è il buco in cui entra il contenuto scritto nel parent. Non usare <Modal> o <Modal.Header> — quella è sintassi React Bootstrap.

modal.tsmodal
@Component({
  selector: 'app-modal',
  imports: [CommonModule],
  templateUrl: './modal.html',
})
export class Modal {
  @Input()  isOpen = false;
  @Input()  title = '';
  @Output() closed = new EventEmitter<void>();
  @Output() confirmed = new EventEmitter<void>();

  close()   { this.closed.emit(); }
  confirm() { this.confirmed.emit(); }
}
modal.htmltemplate
@if (isOpen) {
  <div class="modal d-block">
    <div class="modal-dialog">
      <div class="modal-content">
        <div class="modal-header">
          <h5>{{title}}</h5>
          <button (click)="close()">✕</button>
        </div>
        <div class="modal-body">
          <ng-content></ng-content>  <!-- buco -->
        </div>
        <div class="modal-footer">
          <button (click)="confirm()">Salva</button>
        </div>
      </div>
    </div>
  </div>
  <div class="modal-backdrop fade show"></div>
}
data.html — utilizzo nel parent
<!-- il titolo cambia in base al mode -->
<app-modal
  [isOpen]="showModal"
  [title]="modalMode === 'add' ? 'Nuovo Item' : 'Modifica Item'"
  (closed)="showModal = false"
  (confirmed)="saveItem()">

  <!-- questo entra nel ng-content (modal-body) -->
  @if (selectedItem) {
    <input class="form-control" [(ngModel)]="selectedItem.name" />
  } @else {
    <p>Nuovo item</p>
  }

</app-modal>
data.html └─ <app-modal> <p>contenuto</p> </app-modal>
modal.html └─ <ng-content> ← qui appare il <p> di data.html
📌 Content Projection
Questo meccanismo si chiama Content Projection. È l'equivalente Angular dei children in React o degli slot in Vue. Il modal è generico — non sa cosa ci sarà dentro.
⚠️ Non fare
public modal = new Modal() — i componenti Angular non si istanziano mai con new.
