Angular — @Input/@Output e Parent-Child Binding
Passaggio dati tra componenti, EventEmitter, two-way binding, getter/setter
Sintesi — Regola Mentalequick ref
// [prop]      → parent → child   (Input: riceve dato)
// (propChange) → child → parent  (Output: emette evento)
// [(prop)]    → two-way binding  (Input + Output insieme)
Errore comune
@Output() public myVar: string = '' — sbagliato! @Output() non serve a esportare variabili ma solo a dichiarare EventEmitter. Senza emissione esplicita, il parent non riceverà mai nulla.
Architettura Parent-Child — Schemapattern
        Parent Component
  +-------------------------------+
  | selectedEditProviderId        |  ← proprietà del parent
  | configuration                 |  ← oggetto condiviso
  +-------------------------------+
                 |
                 | [configuration]          → parent → child
                 | [(selectedProviderId)]   → two-way binding
                 v
        Child Component (Edit)
  +-------------------------------+
  | @Input()  configuration       |
  | @Input()  selectedProviderId  |
  | @Output() selectedProviderIdChange = new EventEmitter()
  |                               |
  | onProviderChanged(id)         |
  |  → emit → aggiorna parent    |
  +-------------------------------+
1. @Input() — Parent → Childinput
// Parent
@Component({
  selector: 'app-parent',
  template: `
    <dynamic-component-title-edit
        [description]="description">
    </dynamic-component-title-edit>
  `
})
export class ParentComponent {
  description = 'Titolo iniziale';
}

// Child
@Component({
  selector: 'dynamic-component-title-edit',
  template: `<input [value]="description">`
})
export class DynamicComponentTitleEditComponent {
  @Input() description!: string;
}
2. @Output() + EventEmitter — Child → Parentoutput
// Child: emette l'evento quando il valore cambia
@Component({
  selector: 'dynamic-component-title-edit',
  template: `
    <input [value]="description"
           (input)="onDescriptionChange($event.target.value)">
  `
})
export class DynamicComponentTitleEditComponent {
  @Input()  description!: string;
  @Output() descriptionChange = new EventEmitter<string>();

  onDescriptionChange(value: string) {
    this.descriptionChange.emit(value);
  }
}

// Parent: ascolta l'evento
@Component({
  selector: 'app-parent',
  template: `
    <dynamic-component-title-edit
        [description]="description"
        (descriptionChange)="description = $event">
    </dynamic-component-title-edit>
  `
})
export class ParentComponent {
  description = 'Titolo iniziale';
}
Come funziona il two-way binding [(prop)]
Per abilitare [(description)], il child deve avere sia @Input() description che @Output() descriptionChange (stesso nome + "Change"). Angular espande automaticamente [(description)]="val" in [description]="val" (descriptionChange)="val = $event".
3. Getter/Setter nel Parent — Aggiornamento Oggetto Condivisogetter/setter
// Parent: usa getter/setter per aggiornare configuration
@Component({
  selector: 'app-parent',
  template: `
    <dynamic-component-title-edit
        [(description)]="description">
    </dynamic-component-title-edit>
  `
})
export class ParentComponent {
  configuration: TitleConfiguration = {
    descriptions: [{ languageId: 1, text: 'Titolo iniziale' }],
    headingStyle: 'h1'
  };

  get description(): string {
    return this.configuration.descriptions[0]?.text ?? '';
  }

  set description(value: string) {
    this.configuration.descriptions[0].text = value;
  }
}

Il child usa [(ngModel)]="description": richiama il getter per leggere e il setter per aggiornare l'oggetto configuration nel parent in modo trasparente.

4. Template Parent — Mostrare Edit o View con *ngIftemplate
<!-- Parent template -->
<dynamic-component-checkbox-edit *ngIf="editService.editMode"
    [configuration]="configuration"
    [availableProviders]="availableProviders"
    [(selectedProviderId)]="selectedEditProviderId">
</dynamic-component-checkbox-edit>

<dynamic-component-checkbox-view *ngIf="!editService.editMode"
    [configuration]="configuration">
</dynamic-component-checkbox-view>

  </button>
