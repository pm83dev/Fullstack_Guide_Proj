NGXS — Effects & Actions Stream
ofActionSuccessful, ofActionDispatched, ofActionErrored — reagire agli eventi dello store fuori dallo State
Cos'è il flusso Actionsconcetto

NGXS espone un stream globale di tutte le Action dispatchate tramite il servizio Actions. Puoi iscriverti per reagire a eventi fuori dallo State — notifiche, navigazione, aggiornamento di un altro State.

OperatoreQuando scatta
ofActionDispatchedAppena l'Action viene dispatchata (prima dell'esecuzione)
ofActionSuccessfulDopo che lo State l'ha gestita senza errori
ofActionErroredSe il gestore ha lanciato un'eccezione
Effect — Esempio Completoservice
@Injectable({ providedIn: 'root' })
export class PlantEffects implements OnDestroy {

  private destroy$ = new Subject<void>();

  constructor(
    private actions$: Actions,
    private router:   Router,
    private snackbar: MatSnackBar
  ) {
    // Mostra notifica dopo aggiunta riuscita
    this.actions$.pipe(
      ofActionSuccessful(AddVariable),
      takeUntil(this.destroy$)
    ).subscribe(() => {
      this.snackbar.open('Variabile aggiunta', 'OK', { duration: 3000 });
    });

    // Naviga alla home dopo un errore critico
    this.actions$.pipe(
      ofActionErrored(LoadVariables),
      takeUntil(this.destroy$)
    ).subscribe(() => {
      this.router.navigate(['/']);
    });
  }

  ngOnDestroy() {
    this.destroy$.next();
    this.destroy$.complete();
  }
}
Aggiornare un Secondo State da un Effectcross-state
// Quando LoadVariables ha successo, aggiorna il log state
this.actions$.pipe(
  ofActionSuccessful(LoadVariables),
  takeUntil(this.destroy$)
).subscribe(() => {
  this.store.dispatch(new AddLogEntry({
    action: 'LoadVariables',
    timestamp: new Date()
  }));
});
Quando usare gli Effects
Usa gli Effects per side effect che non appartengono allo State: notifiche UI, navigazione, logging, aggiornamento di altri State. Non mettere logica business negli Effects.
ofActionDispatched — Use Case Spinner GlobaleUI
// Mostra uno spinner globale mentre qualsiasi action asincrona è in corso
constructor(private actions$: Actions, private store: Store) {

  this.actions$.pipe(
    ofActionDispatched(LoadVariables, LoadAlarms, LoadDocuments),
    takeUntil(this.destroy$)
  ).subscribe(() => {
    this.store.dispatch(new SetGlobalLoading(true));
  });

  this.actions$.pipe(
    ofActionSuccessful(LoadVariables, LoadAlarms, LoadDocuments),
    takeUntil(this.destroy$)
  ).subscribe(() => {
    this.store.dispatch(new SetGlobalLoading(false));
  });
}
ctx.dispatch — Concatenare Actions dallo Statestate
// Dentro uno @Action, dispatcha altre Actions come side effect
@Action(AddVariable)
add(ctx: StateContext<PlantStateModel>, { payload }: AddVariable) {
  return this.api.addVariable(payload).pipe(
    tap(newVar => {
      const { variables } = ctx.getState();
      ctx.patchState({ variables: [...variables, newVar] });

      // Dispatcha un'altra action per notificare il log
      ctx.dispatch(new LogActions.AddLogEntry({
        type: 'ADD_VARIABLE',
        data: newVar,
        timestamp: new Date().toISOString()
      }));
    })
  );
}
ctx.dispatch vs Effects — quando usare quale
Usa ctx.dispatch quando la seconda action dipende dal risultato della prima e deve avvenire nello stesso flusso. Usa gli Effects per side effect UI (notifiche, navigazione) che non influenzano lo stato.
