1 — Navigazione e Filesystemnavigazione
ComandoCosa fa
cd ~Vai alla home (/home/nomeutente)
cd -Torna alla cartella precedente
cd ../..Sali di due livelli
ls -laLista dettagliata con file nascosti
ls -lhDimensioni leggibili (KB, MB, GB)
mkdir -p a/b/cCrea cartelle annidate in un colpo solo
cp -r src/ dest/Copia cartella intera (ricorsivo)
mv file.txt ~/altra/Sposta un file in un'altra cartella
rm -rf cartella/Elimina tutto senza conferma — usare con cautela!
find . -name '*.ts'Trova tutti i file .ts nella directory corrente
du -sh cartella/Dimensione totale di una cartella
df -hSpazio disco disponibile su tutti i filesystem
⚠ rm non ha cestino
rm -rf elimina subito e definitivamente — non va nel cestino. Prima di usarlo, verifica sempre il percorso con pwd.
2 — Leggere e Cercare nei Filegrep
ComandoCosa fa
less file.txtVisualizza il file paginato — q per uscire, /testo per cercare
head -n 20 file.txtMostra le prime 20 righe
tail -f app.logSegue il file in tempo reale (fondamentale per i log)
wc -l file.txtConta le righe di un file
grep -i 'error' app.logCerca ignorando maiuscole/minuscole
grep -n 'error' app.logMostra anche il numero di riga
grep -r 'apiUrl' src/Cerca ricorsivamente in tutti i file di una cartella
grep -v 'DEBUG' app.logMostra le righe che NON contengono 'DEBUG'
diff -u file1 file2Differenze in formato unificato (come git diff)
tail -f — il comando per i log
Rimane in ascolto e stampa ogni nuova riga appena viene scritta. Usalo per monitorare log di .NET, Nginx o qualsiasi servizio. Ctrl+C per uscire.
3 — Pipe, Redirect e Composizionepipe
# Pipe: prende output di un comando e lo passa al successivo
ls -la | grep '.ts'
cat app.log | grep 'ERROR' | wc -l
ps aux | grep dotnet

# Redirect: > sovrascrive, >> aggiunge in coda
dotnet build > build-output.txt 2>&1
echo "$(date): avvio" >> deploy.log

# Esecuzione condizionale
dotnet build && dotnet run   # cmd2 solo se cmd1 ha successo
ng serve &                   # avvia in background

5 — Processi e Porteprocessi
ComandoCosa fa

6 — Rete, curl e Trasferimento Filerete
# Test API dal terminale
curl http://localhost:5035/hello
curl -s http://localhost:5035/hello | jq .   # JSON formattato
curl -X POST -H 'Content-Type: application/json' \
     -d '{"nome":"paolo"}' http://localhost:5035/api

# Con JWT
curl -X GET http://localhost:5035/api/dati \
     -H 'Authorization: Bearer eyJ...' | jq .

# Trasferire file sul server
scp -r dist/ user@host:/var/www/
rsync -avz ./dist/ user@host:/var/www/  # solo file modificati
rsync vs scp per i deploy
Per copiare il build Angular sul server, rsync è molto più efficiente di scp: trasferisce solo i file modificati. Con connessioni lente fa una differenza enorme.

8 — Git da Terminale Linuxgit
ComandoCosa fa
git add -pAggiunge interattivamente — scegli chunk per chunk
git log --oneline -10Ultimi 10 commit compatti
git diff --stagedModifiche già in stage
git checkout -b feature/loginCrea e passa a un nuovo branch
git stash / git stash popSalva/ripristina modifiche temporanee
git restore .Annulla tutte le modifiche non committate — irreversibile
git reset --soft HEAD~1Annulla l'ultimo commit ma mantiene le modifiche
Ctrl+R — ricerca nella history
Premi Ctrl+R e inizia a digitare parte di un comando: bash trova l'ultimo corrispondente. È probabilmente il shortcut più utile del terminale.
