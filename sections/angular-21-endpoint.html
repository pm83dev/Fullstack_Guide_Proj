Angular 21 — Chiamare un Endpoint
Service con Observable, gestione loading/errore, finalize, ChangeDetectorRef
Sintesi — Regola Mentalequick ref
// Service    → ritorna sempre Observable, mai Promise
// Componente → subscribe() + finalize() + cdr.detectChanges()
// Mai usare firstValueFrom() / async-await con HttpClient
Perché no async/await?
I callback di Promise girano fuori dalla zona Angular: la proprietà cambia in memoria, ma il DOM non si aggiorna. Con finalize() + cdr.detectChanges() sei tu a controllare esplicitamente il re-render.
1. Service — ritorna Observable, mai Promiseservice
import { HttpClient } from "@angular/common/http";
import { Injectable } from "@angular/core";
import { Observable, timeout } from "rxjs";

@Injectable({ providedIn: "root" })
export class MioService {
  constructor(private http: HttpClient) {}

  getDati(): Observable<MioDato[]> {
    return this.http
      .get<MioDato[]>("http://server/api/dati")
      .pipe(timeout(8000));   // sempre: evita che rimanga in pending
  }
}
2. Componente — 4 ingredienticomponente
import { ChangeDetectorRef, Component } from "@angular/core";
import { Subscription } from "rxjs";
import { finalize } from "rxjs/operators";

@Component({ ... })
export class MioComponent {
  isBusy = false;
  dati: MioDato[] = [];
  errore = "";

  private sub?: Subscription;        // ① per fare unsubscribe

  constructor(
    private svc: MioService,
    private cdr: ChangeDetectorRef,  // ② forza il re-render
  ) {}

  ngOnDestroy() {
    this.sub?.unsubscribe();          // ③ pulizia se l'utente naviga via
  }

  carica() {
    this.isBusy = true;
    this.errore = "";

    this.sub = this.svc.getDati()
      .pipe(
        finalize(() => {              // ④ equivalente di finally — gira sempre
          this.isBusy = false;
          this.cdr.detectChanges();  // ⑤ forza Angular a vedere i cambiamenti
        }),
      )
      .subscribe({
        next:  (dati) => (this.dati   = dati),
        error: (err)  => (this.errore = "Errore: " + err.message),
      });
  }
}

Perché NON usare async/await?anti-pattern
// ❌ NON fare così in Angular 21
async carica() {
  this.isBusy = true;
  try {
    this.dati = await firstValueFrom(this.svc.getDati());
  } finally {
    this.isBusy = false;  // viene eseguito, ma Angular non lo vede
  }
}
Il problema
Il finally viene eseguito fuori dalla zona Angular → la proprietà cambia in memoria, ma il DOM non si aggiorna. Con cdr.detectChanges() nel finalize di un Observable sei tu a controllare esplicitamente quando Angular ri-renderizza.
