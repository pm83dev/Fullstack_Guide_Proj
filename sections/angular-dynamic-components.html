Angular — Componenti Dinamici con Multi-Provider
InjectionToken con multi:true, DynamicComponentsService, createComponent(), getComponents()
Come Funziona — Schema Completopattern
Registrazione componenti        (providers con multi:true)
        ↓
DynamicComponentsService        (iniettato tramite token)
        ↓
getComponents()                 (restituisce lista filtrata)
        ↓
allComponents nel container     (getter del componente)
        ↓
*ngFor="let c of allComponents" (template)
        ↓
addComponent(c) → createComponent() → container.createComponent()
Perché multi:true?
Con multi: true, Angular raccoglie tutte le registrazioni con lo stesso token in un array e lo inietta come tale. Ogni modulo/feature può registrare i propri componenti indipendentemente, senza toccare il codice centrale.
1. Definizione del Tokentoken
import { InjectionToken } from '@angular/core';

export interface DynamicComponentProvider {
  key: string;
  type: any;         // la classe del componente Angular
  metadata?: any;    // dati extra (label, icona, ecc.)
}

export const ACCREDITATION_DYNAMIC_COMPONENT_PROVIDER =
  new InjectionToken<DynamicComponentProvider[]>(
    'ACCREDITATION_DYNAMIC_COMPONENT_PROVIDER'
  );
2. Registrazione dei Componenti (multi:true)providers
// In un modulo o in providers di una route standalone
providers: [
  {
    provide: ACCREDITATION_DYNAMIC_COMPONENT_PROVIDER,
    multi: true,
    useValue: {
      key: 'textbox',
      type: TextBoxComponent,
      metadata: { label: 'Campo Testo' }
    }
  },
  {
    provide: ACCREDITATION_DYNAMIC_COMPONENT_PROVIDER,
    multi: true,
    useValue: {
      key: 'table',
      type: TableComponent,
      metadata: { label: 'Tabella Dati' }
    }
  },
  // ogni feature aggiunge la propria voce senza toccare le altre
]
Senza questo, getComponents() restituisce lista vuota
Se dimentichi di registrare il provider con multi: true, il servizio riceve un array vuoto e nessun componente dinamico viene mostrato.
3. DynamicComponentsServiceservice
@Injectable({ providedIn: 'root' })
export class DynamicComponentsService {

  constructor(
    @Inject(ACCREDITATION_DYNAMIC_COMPONENT_PROVIDER)
    private readonly _componentsProvider: DynamicComponentProvider[]
  ) {}

  // Restituisce tutti i componenti, o filtrati
  getComponents(
    filter?: (c: DynamicComponentProvider) => boolean
  ): DynamicComponentProvider[] {
    return filter
      ? this._componentsProvider.filter(filter)
      : this._componentsProvider;
  }

  // Trova il provider per un tipo specifico
  private getComponent(type: string): DynamicComponentProvider {
    const found = this._componentsProvider.find(c => c.key === type);
    if (!found) throw new Error(`Componente "${type}" non registrato`);
    return found;
  }

  // Crea il componente nel ViewContainerRef
  createComponent(
    container: ViewContainerRef,
    definition: { type: string; [key: string]: any }
  ) {
    const componentData = this.getComponent(definition.type);
    const componentRef  = container.createComponent(componentData.type);

    // Passa la definizione al componente dinamico
    const instance = componentRef.instance as DynamicComponent;
    instance.loadDefinition(definition);

    return { componentRef, dynamicComponent: instance, metadata: componentData.metadata };
  }
}
4. Uso nel Componente Containercontainer
@Component({
  selector: 'accreditation-components-container',
  template: `
    <div *ngFor="let c of allComponents">
      <button (click)="addComponent(c)">
        {{ c.metadata?.label }}
      </button>
    </div>
    <ng-container #host></ng-container>
  `
})
export class AccreditationComponentsContainerComponent {
  @ViewChild('host', { read: ViewContainerRef })
  container!: ViewContainerRef;

  constructor(
    private _dynamicComponentService: DynamicComponentsService,
    public  filter: FilterService   // opzionale
  ) {}

  // Lista di tutti i componenti disponibili (filtrabili)
  public get allComponents(): DynamicComponentProvider[] {
    return this._dynamicComponentService
      .getComponents(this.filter?.isValidItem);
  }

  addComponent(provider: DynamicComponentProvider): void {
    this._dynamicComponentService
      .createComponent(this.container, { type: provider.key });
  }
}
