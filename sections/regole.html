Regole d'oro & Best Practices
Le regole che si applicano sempre, a prescindere dal contesto
🔶 Regola Fastkart
Se un componente non fa dispatch, non deve conoscere il filtro.
🔶 ngModel
[(ngModel)] solo per proprietà semplici. Tutto il resto → [ngModel] + (ngModelChange).
🔶 Stato UI
Lo stato UI (showModal, isLoading...) NON deve mai essere serializzato nello State NGXS.
🔶 @Input/@Output
In Angular i componenti si comunicano tramite @Input/@Output, non tramite props JSX. Il selettore è sempre quello definito in @Component({ selector: '...' }).
🔶 Non istanziare componenti con new
public modal = new Modal() è sbagliato. I componenti Angular li gestisce il framework, non il codice.
