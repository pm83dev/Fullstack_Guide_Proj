RxJS — Operatori Avanzati
map, switchMap, takeUntil, ofActionSuccessful — quando e perché
Operatori in un colpo d'occhio
OperatoreCosa faQuando usarlo
mapTrasforma ogni valore che passaFiltrare array, cambiare forma del dato, conteggi
tapSide effect senza toccare il valorepatchState, console.log, aggiornare stato loading
catchErrorIntercetta errore, ritorna Observable alternativoSempre negli @Action asincroni
switchMapRiceve valore → ritorna nuovo Observable, cancella il precedenteIncatenare HTTP, ricerche live (annulla la precedente)
takeUntilCompleta il flusso quando un altro Observable emetteUnsubscribe automatico al destroy del componente
of(value)Observable che emette subito e completaRitorno di fallback in catchError
map — trasforma il valore
this.api.getVariables().pipe(
  map(vars => vars.filter(v => v.active === true)), // tieni solo le attive
  map(vars => vars.length) // solo il conteggio
)
// Observable<OpcVariable[]> → Observable<number>
switchMap — incatena HTTP calls
// Scenario: seleziono un plant → carico le sue variabili
this.store.select(PlantState.getSelectedId).pipe(
  switchMap(id => this.api.getVariablesByPlant(id))
)
// Se arriva un nuovo id prima che finisca la prima chiamata,
// switchMap CANCELLA la precedente e parte con la nuova. ✅
takeUntil — unsubscribe automatico
private destroy$ = new Subject<void>();

ngOnInit() {
  this.store.select(PlantState.getVariables).pipe(
    takeUntil(this.destroy$) // si disiscrive quando destroy$ emette
  ).subscribe(vars => this.doSomething(vars));
}

ngOnDestroy() {
  this.destroy$.next();
  this.destroy$.complete();
}
// Alternativa moderna: takeUntilDestroyed() da @angular/core/rxjs-interop
ofActionSuccessful — reagire fuori dallo State
// Nel costruttore di un service / effects class
this.actions$.pipe(
  ofActionSuccessful(AddVariable),
  takeUntil(this.destroy$)
).subscribe(() => {
  this.snackbar.open('Variabile aggiunta', 'OK', { duration: 3000 });
});
// Usa per: notifiche UI, navigazione, logging — NON per logica di business
VarianteQuando scatta
ofActionDispatchedAppena l'Action viene dispatchata (prima dell'esecuzione)
ofActionSuccessfulDopo che lo State l'ha gestita senza errori
ofActionErroredSe il gestore ha lanciato un'eccezione
