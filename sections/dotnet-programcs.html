⚠️ Ordine middleware critico
La sequenza corretta è: UseHttpsRedirection → UseCors → UseAuthentication → UseAuthorization → MapControllers. Invertire UseAuthentication e UseAuthorization causa errori 401 inspiegabili.
Program.cs — Scheletro completoC#
var builder = WebApplication.CreateBuilder(args);

// ─── Servizi ──────────────────────────────────────────
builder.Services.AddControllers();
builder.Services.AddEndpointsApiExplorer();
builder.Services.AddSwaggerGen();

// DbContext
builder.Services.AddDbContext<AppDbContext>(opt =>
    opt.UseSqlServer(builder.Configuration.GetConnectionString("Default")));

// CORS (solo in sviluppo)
builder.Services.AddCors(opt => opt.AddPolicy("Dev", p =>
    p.WithOrigins("http://localhost:4200").AllowAnyMethod().AllowAnyHeader()));

// JWT
builder.Services.AddAuthentication(JwtBearerDefaults.AuthenticationScheme)
    .AddJwtBearer(opt => {
        opt.TokenValidationParameters = new TokenValidationParameters {
            ValidateIssuerSigningKey = true,
            IssuerSigningKey = new SymmetricSecurityKey(
                Encoding.UTF8.GetBytes(builder.Configuration["Jwt:Key"]!)),
            ValidateIssuer = false,
            ValidateAudience = false
        };
    });

// DI custom
builder.Services.AddScoped<IProductRepository, ProductRepository>();

var app = builder.Build();

// ─── Pipeline ─────────────────────────────────────────
if (app.Environment.IsDevelopment()) {
    app.UseSwagger(); app.UseSwaggerUI();
}

app.UseCors("Dev");          // ← PRIMA di MapControllers
app.UseAuthentication();     // ← PRIMA di UseAuthorization
app.UseAuthorization();
app.MapControllers();
app.Run();
Flusso builder / app
builder.Services.Add*()→Registra servizi nella DI
builder.Build()→Punto di separazione — dopo: niente più Add*()
app.Use*()→Configura pipeline middleware (ordine obbligatorio)
app.Map*()→Mappa endpoint (controllers, minimal API)
app.Run()→Avvia il server HTTP
appsettings.json — sezioni comuniJSON
{
  "ConnectionStrings": {
    "Default": "Server=.;Database=MyDb;Trusted_Connection=True;TrustServerCertificate=True"
  },
  "Jwt": {
    "Key": "una-chiave-segreta-lunga-almeno-32-caratteri"
  },
  "Logging": { "LogLevel": { "Default": "Information" } },
  "AllowedHosts": "*"
}

⚠️ Non committare mai segreti reali (chiavi JWT, password DB) in appsettings.json. In sviluppo usa dotnet user-secrets, in produzione usa variabili d'ambiente o Azure Key Vault.

Pacchetti NuGet essenziali
# Entity Framework Core (SQL Server)
dotnet add MyProject.Infrastructure package Microsoft.EntityFrameworkCore.SqlServer
dotnet add MyProject.Infrastructure package Microsoft.EntityFrameworkCore.Tools

# JWT Auth
dotnet add MyProject.Api package Microsoft.AspNetCore.Authentication.JwtBearer

# Swagger
dotnet add MyProject.Api package Swashbuckle.AspNetCore

# AutoMapper (opzionale)
dotnet add MyProject.Api package AutoMapper.Extensions.Microsoft.DependencyInjection

⚠️ EntityFrameworkCore.Tools va su Infrastructure, non su Api. Se installato nel posto sbagliato, dotnet ef non trova il DbContext.
