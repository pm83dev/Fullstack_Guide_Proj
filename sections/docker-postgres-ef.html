Docker — PostgreSQL + EF Core Migrations
Container PostgreSQL locale, migrazioni EF Core, AppDbContextFactory, comandi psql
Sequenza Completa — Da Zero a DB Funzionanteworkflow
# 1. Avvia PostgreSQL in Docker
docker run --name dpi-postgres-test \
  -e POSTGRES_PASSWORD=postgres \
  -e POSTGRES_DB=DpiDb \
  -p 5432:5432 \
  -d postgres:15-alpine

# 2. Crea la migrazione
dotnet ef migrations add InitialCreate \
  --project dpi.infrastructure \
  --startup-project dpi.backend \
  --connection "Host=localhost;Database=DpiDb;Username=postgres;Password=postgres"

# 3. Applica la migrazione al database
dotnet ef database update \
  --project dpi.infrastructure \
  --startup-project dpi.backend \
  --connection "Host=localhost;Database=DpiDb;Username=postgres;Password=postgres"

# 4. Verifica le tabelle create
docker exec -it dpi-postgres-test psql -U postgres -d DpiDb -c "\dt"

# 5. Avvia l'app
cd dpi.backend && dotnet run

Nota sui nomi colonna PostgreSQL
PostgreSQL è case-sensitive sui nomi delle colonne. Se EF Core genera colonne con maiuscole (es. "FirstName"), vanno sempre messe tra doppi apici nei comandi SQL manuali.

AppDbContextFactory.cs — Per i Comandi EF da Terminalemigrations

Necessaria quando si eseguono migrazioni da terminale (dotnet ef migrations add) in un progetto multi-layer dove il DbContext è in un progetto separato dall'API.

using Microsoft.EntityFrameworkCore;
using Microsoft.EntityFrameworkCore.Design;
using Microsoft.Extensions.Configuration;

public class AppDbContextFactory : IDesignTimeDbContextFactory<AppDbContext>
{
    public AppDbContext CreateDbContext(string[] args)
    {
        var basePath = Directory.GetCurrentDirectory();
        var apiSettingsPath = Path.Combine(
            basePath, "..", "AppBase.Api", "appsettings.Development.json"
        );
        string connectionString = null;

        if (File.Exists(apiSettingsPath))
        {
            var config = new ConfigurationBuilder()
                .SetBasePath(Path.GetFullPath(
                    Path.Combine(basePath, "..", "AppBase.Api")))
                .AddJsonFile("appsettings.Development.json", optional: false)
                .Build();
            connectionString = config.GetConnectionString("DefaultConnection");
        }

        // Fallback se il file di config non è raggiungibile
        if (string.IsNullOrWhiteSpace(connectionString))
        {
            connectionString = "Host=localhost;Database=AppBaseDb;Username=postgres;Password=postgres";
        }

        var optionsBuilder = new DbContextOptionsBuilder<AppDbContext>();
        optionsBuilder.UseNpgsql(connectionString);
        return new AppDbContext(optionsBuilder.Options);
    }
}
Quando serve AppDbContextFactory
EF Core cerca una factory a design-time quando il DbContext è in un progetto diverso dallo startup project (es. architettura clean con Infrastructure separato). Senza di essa, dotnet ef migrations add fallisce con errore "Unable to create DbContext".
Comandi EF Core — Riferimento Rapidoquick ref
# Progetto singolo (DbContext nello stesso progetto dell'API)
dotnet ef migrations add InitialCreate
dotnet ef database update

# Progetto multi-layer (DbContext in Infrastructure, API separata)
dotnet ef migrations add InitialCreate \
  --project dpi.infrastructure \
  --startup-project dpi.backend

# Con connection string esplicita (override appsettings)
dotnet ef database update \
  --project dpi.infrastructure \
  --startup-project dpi.backend \
  --connection "Host=localhost;Database=DpiDb;Username=postgres;Password=postgres"

# Rollback — torna a una migrazione precedente
dotnet ef database update NomeMigrazioneTarget

# Lista migrazioni applicate
dotnet ef migrations list

# Rimuovere l'ultima migrazione (se non ancora applicata al DB)
dotnet ef migrations remove
