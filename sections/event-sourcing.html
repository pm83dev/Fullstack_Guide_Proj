Event Sourcing — Pattern e Implementazione
Event, AggregateRoot, CommandHandler, EventStore generico — basato su WebDpi.Application
Cos'è l'Event Sourcingconcetto
Idea centrale
Invece di salvare solo lo stato corrente di un'entità, si salva la sequenza di eventi che l'hanno portata in quello stato. Lo stato attuale si ricostruisce riproducendo gli eventi in ordine. Ogni operazione (create, update, delete) produce un evento immutabile con timestamp.
Vantaggi
Audit trail completo e gratuito. Possibilità di ricostruire lo stato a qualsiasi punto nel tempo. Disaccoppiamento tra scrittura (comandi) e lettura (query). Facilita event-driven architecture e CQRS.

1. IEntity — Contratto per le Entitàinterfaccia
// Models/IEntity.cs
public interface IEntity
{
    Guid Id { get; set; }
}

Tutte le entità gestite dall'event store implementano questa interfaccia. Il Guid Id diventa il StreamId: la chiave che raggruppa tutti gli eventi relativi a quell'entità.

2. Event — Classe Base Immutabileevento
// Events/Event.cs
public abstract class Event
{
    public Guid     EventId   { get; init; }  // ID univoco dell'evento
    public Guid     StreamId  { get; init; }  // ID dell'entità a cui è legato
    public string   EventType { get; init; }  // "Created" | "Updated" | "Deleted"
    public DateTime CreatedAt { get; init; } = DateTime.UtcNow;
    public object   Data      { get; init; }  // snapshot dell'entità al momento dell'evento

    protected Event(Guid streamId, string eventType, object data)
    {
        StreamId  = streamId;
        EventType = eventType;
        Data      = data;
        EventId   = Guid.NewGuid(); // ogni evento ha ID unico
    }
}
init invece di set
init (C# 9+) rende le proprietà immutabili dopo la costruzione: si possono valorizzare solo nel costruttore o in un object initializer. Gli eventi non devono mai cambiare dopo essere stati creati.
3. Tipi di Evento Concretieventi
// CreatedEvent — genera nuovo StreamId
public class CreatedEvent : Event
{
    public CreatedEvent(object data)
        : base(Guid.NewGuid(), "Created", data) { }
}

// UpdatedEvent — usa l'ID esistente dell'entità
public class UpdatedEvent : Event
{
    public UpdatedEvent(Guid entityId, object updatedData)
        : base(entityId, "Updated", updatedData) { }
}

// DeletedEvent — stesso ID, marca come eliminato
public class DeletedEvent : Event
{
    public DeletedEvent(Guid entityId, object deleteData)
        : base(entityId, "Deleted", deleteData) { }
}
Created genera un nuovo GUID, Update/Delete riusano quello esistente
Il StreamId è la chiave che collega tutti gli eventi della stessa entità. La creazione genera un nuovo GUID; tutte le operazioni successive usano lo stesso ID per formare lo "stream" di eventi.
4. AggregateRoot — Buffer degli Uncommitted Eventsaggregate
public class AggregateRoot
{
    public Guid     streamId  { get; private set; }
    public DateTime CreatedAt { get; private set; }

    private readonly List<Event> _uncommittedEvents = new();
    public IEnumerable<Event> UncommittedEvents => _uncommittedEvents;

    // Applica l'evento e lo aggiunge alla lista uncommitted
    public void ApplyEvent(Event @event)
    {
        switch (@event)
        {
            case CreatedEvent e:  Apply(e); break;
            case UpdatedEvent e:  Apply(e); break;
            case DeletedEvent e:  Apply(e); break;
            default: throw new ArgumentException($"Tipo evento non gestito: {@event.GetType().Name}");
        }
        _uncommittedEvents.Add(@event);
    }

    private void Apply(CreatedEvent e) { streamId = e.StreamId; CreatedAt = e.CreatedAt; }
    private void Apply(UpdatedEvent e) { streamId = e.StreamId; CreatedAt = e.CreatedAt; }
    private void Apply(DeletedEvent e) { streamId = e.StreamId; CreatedAt = e.CreatedAt; }

    // Dopo il salvataggio: svuota il buffer
    public void ClearUncommittedEvents() => _uncommittedEvents.Clear();
}
Perché il buffer UncommittedEvents?
Il pattern permette di applicare più eventi all'aggregato e poi salvarli tutti insieme in modo transazionale. Finché non chiami SaveEventStoreAsync, gli eventi sono solo in memoria. Dopo il salvataggio si chiama ClearUncommittedEvents() per resettare.
5. IEventStore<T> — Contratto Genericointerfaccia
public interface IEventStore<TEntity> where TEntity : IEntity
{
    // Salva un singolo evento
    Task SaveEventStoreAsync(Event @event);

    // Restituisce l'ultimo stato di tutte le entità
    Task<IEnumerable<TEntity>> GetAllItem();

    // Restituisce la cronologia completa di un'entità
    Task<IEnumerable<Event>> GetEventsByIdAsync(Guid streamId);
}
6. InMemoryEventStore<T> — Implementazione In Memoriaimplementazione
public class InMemoryEventStore<TEntity> : IEventStore<TEntity>
    where TEntity : class, IEntity
{
    // Dizionario: StreamId → lista ordinata di eventi
    private readonly Dictionary<Guid, List<Event>> _eventStore = new();

    public Task SaveEventStoreAsync(Event @event)
    {
        if (!_eventStore.ContainsKey(@event.StreamId))
            _eventStore.Add(@event.StreamId, new List<Event>());

        _eventStore[@event.StreamId].Add(@event);
        return Task.CompletedTask;
    }

    // Ricostruisce lo stato corrente: prende l'ultimo evento per ogni StreamId
    public Task<IEnumerable<TEntity>> GetAllItem()
    {
        var entities = _eventStore.Values
            .SelectMany(events => events)
            .GroupBy(e => e.StreamId)
            .Select(group => group
                .OrderByDescending(e => e.CreatedAt)
                .FirstOrDefault())          // ultimo evento = stato corrente
            .Select(@event => @event?.Data as TEntity)
            .Where(entity => entity != null)
            .Select(entity => entity!)
            .ToList();

        return Task.FromResult<IEnumerable<TEntity>>(entities);
    }

    // Restituisce TUTTI gli eventi di un'entità (cronologia completa)
    public Task<IEnumerable<Event>> GetEventsByIdAsync(Guid streamId)
    {
        if (!_eventStore.ContainsKey(streamId))
            return Task.FromResult(Enumerable.Empty<Event>());

        return Task.FromResult(_eventStore[streamId].AsEnumerable());
    }
}
7. CommandHandler<T> — Orchestratorecommand
public class CommandHandler<TEntity> : ICommands<TEntity>
    where TEntity : IEntity
{
    private readonly IEventStore<TEntity> _eventStore;

    public CommandHandler(IEventStore<TEntity> eventStore)
        => _eventStore = eventStore;

    public async Task HandleCreateAsync(TEntity entity)
    {
        if (entity.Id == Guid.Empty)
            entity.Id = Guid.NewGuid(); // assegna ID se mancante

        var @event = new CreatedEvent(entity) { StreamId = entity.Id };

        var aggregate = new AggregateRoot();
        aggregate.ApplyEvent(@event);

        foreach (var e in aggregate.UncommittedEvents)
            await _eventStore.SaveEventStoreAsync(e);

        aggregate.ClearUncommittedEvents();
    }

    public async Task HandleUpdateAsync(TEntity entity)
    {
        var @event = new UpdatedEvent(entity.Id, entity);

        var aggregate = new AggregateRoot();
        aggregate.ApplyEvent(@event);

        foreach (var e in aggregate.UncommittedEvents)
            await _eventStore.SaveEventStoreAsync(e);

        aggregate.ClearUncommittedEvents();
    }

    public async Task HandleDeleteAsync(TEntity entity)
    {
        var @event = new DeletedEvent(entity.Id, entity);

        var aggregate = new AggregateRoot();
        aggregate.ApplyEvent(@event);

        foreach (var e in aggregate.UncommittedEvents)
            await _eventStore.SaveEventStoreAsync(e);

        aggregate.ClearUncommittedEvents();
    }
}
8. Controller — Endpoint Create / Update / GetEventscontroller
[ApiController]
[Route("[controller]")]
public class UtensiliController : ControllerBase
{
    private readonly ICommands<Utensile>   _commandHandler;
    private readonly IEventStore<Utensile> _eventStore;
    private readonly IUtensiliService      _service;

    // POST /Utensili/CreateEvent
    [HttpPost("CreateEvent")]
    public async <IActionResult> AddDpiEvent(Utensile utensile)
    {
        await _commandHandler.HandleCreateAsync(utensile);
        return Ok(utensile);
    }

    // PUT /Utensili/UpdateEvent/{id}
    [HttpPut("UpdateEvent/{dpi}")]
    public async <IActionResult> UpdateDpiEvent(Utensile utensile)
    {
        await _commandHandler.HandleUpdateAsync(utensile);
        return Ok(utensile);
    }

    // GET /Utensili/GetAllDpis — stato corrente + eventi store
    [HttpGet("GetAllDpis")]
    public async <IActionResult> GetAllEvents()
    {
        var utensili       = await _service.GetAllUtensili();     // da DB relazionale
        var utensiliEvents = await _eventStore.GetAllItem();     // da event store
        return Ok(new { Utensili = utensili, UtensiliEvents = utensiliEvents });
    }

    // GET /Utensili/{id}/events — cronologia completa di un'entità
    [HttpGet("{dpiId}/events")]
    public async <IActionResult> GetDpiEvents(Guid dpiId)
    {
        var events = await _eventStore.GetEventsByIdAsync(dpiId);
        if (!events.Any()) return NotFound();
        return Ok(events);
    }
}

public static IServiceCollection InjectApplicationServices(
    this IServiceCollection self)
{
    // Registrazione generica open — funziona per qualsiasi TEntity

typeof() per generici open
typeof(IEventStore<>) con le parentesi angolari vuote registra il tipo generico aperto. Angular DI risolve automaticamente IEventStore<Utensile>, IEventStore<Operatore> ecc. senza dover registrare ogni combinazione separatamente.
10. Value Object — Record Immutabili con ValidazioneDDD
// ValueObject/ValueObjectDpi.cs
// Ogni VO è un record con validazione nel costruttore

public record Name
{
    public string Value { get; }

    public Name(string value)
    {
        if (string.IsNullOrWhiteSpace(value))
            throw new ArgumentException("Il nome non può essere vuoto");
        Value = value;
    }

    public override string ToString() => Value;
}

// Stesso pattern per: Surname, Role, Username, Password,
// Email, Phone, Address, Code, Description

// Per date con validazione
public record AssignmentDate
{
    public DateOnly Value { get; }

    public AssignmentDate(DateOnly value)
    {
        // validazione: data non nel futuro
        Value = value;
    }
}
Perché record e non class?
I record in C# hanno uguaglianza strutturale per default (Name("Mario") == Name("Mario")), sono immutabili se usi solo proprietà get, e si stampano bene. Perfetti per Value Objects nel Domain-Driven Design.

Estendere — Da InMemory a MongoDB o PostgreSQLproduzione

Basta implementare IEventStore<T> con un nuovo backend e cambiare la registrazione DI. Il resto del codice non cambia.

// EventStore/MongoEventStore.cs
public class MongoEventStore<TEntity> : IEventStore<TEntity>
    where TEntity : class, IEntity
{
    private readonly IMongoCollection<EventDocument> _collection;

    public async Task SaveEventStoreAsync(Event @event)
    {
        var doc = new EventDocument
        {
            EventId   = @event.EventId,
            StreamId  = @event.StreamId,
            EventType = @event.EventType,
            CreatedAt = @event.CreatedAt,
            Data      = BsonDocument.Parse(JsonSerializer.Serialize(@event.Data))
        };
        await _collection.InsertOneAsync(doc);
    }

    // ... implementa GetAllItem() e GetEventsByIdAsync()
}

// ServiceCollection: cambia solo questa riga
self.AddSingleton(typeof(IEventStore<>), typeof(MongoEventStore<>));
