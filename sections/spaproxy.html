SpaProxy .NET + Angular
Avvio integrato dev: F5 da VS Code lancia sia .NET che Angular CLI
Come funziona
VS Code F5 → Legge launch.json, lancia Chrome su https://localhost:50663
↓
.NET avvia → ASPNETCORE_HOSTINGSTARTUPASSEMBLIES=Microsoft.AspNetCore.SpaProxy
↓
SpaProxy → Lancia npm start → Angular su https://localhost:4200
↓
Proxy → :50663 → Angular :4200 | API REST rimangono su .NET
File: AppBase.Api.csproj
<PropertyGroup>
  <SpaRoot>..\frontend</SpaRoot>
  <SpaProxyServerUrl>http://localhost:4200</SpaProxyServerUrl>
  <SpaProxyLaunchCommand>npm start</SpaProxyLaunchCommand>
  <SpaProxyWorkingDirectory>$(SpaRoot)</SpaProxyWorkingDirectory>
</PropertyGroup>

<ItemGroup>
  <PackageReference Include="Microsoft.AspNetCore.SpaProxy" Version="10.0.5" />
</ItemGroup>
File: launchSettings.json (profilo HTTPS)
"https": {
  "commandName": "Project",
  "launchBrowser": true,
  "environmentVariables": {
    "ASPNETCORE_ENVIRONMENT": "Development",
    "ASPNETCORE_HOSTINGSTARTUPASSEMBLIES": "Microsoft.AspNetCore.SpaProxy"
  },
  "applicationUrl": "https://localhost:7083;http://localhost:5095"
}
File: proxy.conf.js (Angular → .NET)
const { env } = require('process');

const target = env.ASPNETCORE_HTTPS_PORT
  ? `https://localhost:${env.ASPNETCORE_HTTPS_PORT}`
  : env.ASPNETCORE_URLS
    ? env.ASPNETCORE_URLS.split(';')[0]
    : 'https://localhost:7083';

const PROXY_CONFIG = [{
  context: ['/api/v1.0/', '/swagger', '/favicon.ico'],
  target,
  secure: false
}];

module.exports = PROXY_CONFIG;
📌 Punti minimi obbligatori
1. PackageReference per SpaProxy nel .csproj
2. ASPNETCORE_HOSTINGSTARTUPASSEMBLIES nel launchSettings.json
3. launch.json opzionale ma consigliato per F5 coerente da VS Code
