Step 1 — Cartelle per i Volumihost
# Creare cartelle sul VPS
sudo mkdir -p /srv/[nome-app]/images
sudo mkdir -p /srv/[nome-app]/documents

# Assegnare permessi PRIMA di qualsiasi trasferimento
sudo chown -R deployuser:deployuser /srv/[nome-app]

# Verificare
ls -la /srv/[nome-app]/
⚠ Permessi prima del trasferimento
Eseguire sempre il chown PRIMA di trasferire file via WinSCP. Altrimenti si ottiene "Permission denied".
Step 2 — Blocco Servizio in docker-compose-ssl.ymlcompose
# Aprire il file
nano /home/deployuser/docker-compose-ssl.yml

# Aggiungere in fondo alla sezione services:
  [nome-servizio]:
    image: pm83dev/[nome-immagine]:latest
    container_name: [nome-servizio]
    restart: unless-stopped
    environment:
      - DOTNET_ENVIRONMENT=Production
      - ASPNETCORE_URLS=http://+:5000
      - ConnectionStrings__Fashion=Server=sqlserver,1433;Database=[NomeDB];UserId=sa;Password=1.Password!;TrustServerCertificate=true;
    depends_on:
      - sqlserver
    networks:
      - gadis-network
    expose:
      - "5000"
    volumes:
      - /srv/[nome-app]/images:/data/images
      - /srv/[nome-app]/documents:/data/documents

# Verificare sintassi YAML
docker compose -f docker-compose-ssl.yml config
⚠ Doppio underscore ConnectionStrings
ConnectionStrings__Fashion (doppio underscore) sovrascrive automaticamente ConnectionStrings:Fashion nell'appsettings.json. I __ corrispondono ai : nella gerarchia JSON di .NET.
Step 3 — Restore Database SQL Serverdatabase
# Trasferire backup con WinSCP in: /home/deployuser/backups/
# Per file grandi (>2GB) usare rsync da WSL:
rsync -av --progress /mnt/c/percorso/file.back deployuser@IP:/home/deployuser/backups/

# Verificare i nomi logici del backup
docker exec -it sqlserver /opt/mssql-tools18/bin/sqlcmd \
  -S localhost -U sa -P "1.Password!" -C \
  -Q "RESTORE FILELISTONLY FROM DISK = '/backups/[file.back]'"

# Eseguire il restore
docker exec -it sqlserver /opt/mssql-tools18/bin/sqlcmd \
  -S localhost -U sa -P "1.Password!" -C \
  -Q "RESTORE DATABASE [NomeDB] FROM DISK = '/backups/[file.back]'
  WITH MOVE '[LogicalName_dati]' TO '/var/opt/mssql/data/[NomeDB].mdf',
  MOVE '[LogicalName_log]' TO '/var/opt/mssql/data/[NomeDB]_log.ldf', REPLACE"

# Verificare che il database sia presente
docker exec -it sqlserver /opt/mssql-tools18/bin/sqlcmd \
  -S localhost -U sa -P "1.Password!" -C \
  -Q "SELECT name FROM sys.databases"
⚠ WITH MOVE obbligatorio
Il backup ricorda i percorsi originali Windows (C:\Program Files\...) che non esistono nel container Linux. WITH MOVE reindirizza i file ai percorsi corretti.
Step 4-6 — Nginx Config + SSL Certbotnginx
# Step 4 — Crea config HTTP (solo per la verifica Certbot)
nano /home/deployuser/nginx/conf.d/[nome-app].conf
server {
    listen 80;
    server_name [sottodominio.dominio.com];
    location /.well-known/acme-challenge/ {
        root /var/www/certbot;
    }
    location / {
        return 301 https://$host$request_uri;
    }
}
# Ricarica Nginx
docker exec nginx-proxy nginx -s reload

# Step 5 — Genera certificato SSL
docker exec certbot certbot certonly \
  --webroot -w /var/www/certbot \
  -d [sottodominio.dominio.com] \
  --email [email] --agree-tos --no-eff-email
# Step 6 — Config HTTPS completa
server {
    listen 443 ssl;
    server_name [sottodominio.dominio.com];
    ssl_certificate /etc/letsencrypt/live/[sottodominio]/fullchain.pem;
    ssl_certificate_key /etc/letsencrypt/live/[sottodominio]/privkey.pem;
    proxy_http_version 1.1;
    proxy_set_header Connection "";
    location / {
        proxy_pass http://[nome-servizio]:5000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-Proto $scheme;
    }
}
proxy_pass — usa il nome servizio Docker
In proxy_pass usa il nome del servizio Docker (chiave nel compose), non il container_name. Nginx risolve i nomi tramite la rete Docker interna.
Step 8 — Pull Immagine e Avvio Containerdocker
# Pull dell'immagine
docker compose -f docker-compose-ssl.yml pull [nome-servizio]

# Avvio del container (senza toccare gli altri)
docker compose -f docker-compose-ssl.yml up -d [nome-servizio]

# Riavvio forzato
docker compose -f docker-compose-ssl.yml up -d --force-recreate [nome-servizio]

# Ricaricare Nginx dopo l'avvio
docker exec nginx-proxy nginx -s reload

# Verificare log
docker logs [nome-servizio] --tail 50
